## 2. 🔴 观测器算法本身的问题 — 已全部修复

### 问题 1：PLL 积分器没有抗饱和 ✅ 已修复

**原问题**: `PLL_Interg` 无上限，误差大时积分飞掉。

**修复** (`flux.c` → `Observer_Run`):
```c
pll_ui_lim = 2.0f * PI * 500.0f * Foc_observer.Ctrl_ts;  // ±0.157 rad/周期
Foc_observer.PLL_Interg = Limit_Sat(Foc_observer.PLL_Interg, pll_ui_lim, -pll_ui_lim);
Foc_observer.PLL_Ui = Limit_Sat(Foc_observer.PLL_Ui, pll_ui_lim, -pll_ui_lim);
```

### 问题 2：从零冷启动，观测器需要上百毫秒才能收敛 ✅ 已修复

**原问题**: VF→SENSORLESS 直接冷启动，观测器从零开始收敛。

**修复** (`motorControl.c` → Mode 4):
- Mode 4 只做 VF 驱动 + 观测器预热，不自动切换
- 用户确认 LCD "Psi" ≈ 0.0075 后手动 KEY2 切 Mode 2
- Mode 2 检测 `|ψr| ≥ 30% FLUX` → 暖启动（PLL 不动，直接接管）

### 问题 3：低转速时逆变器死区造成的电压误差被忽略 ✅ 已修复

**修复** (`flux.c` → `Deadtime_Comp_ab`):
- 命令电压 × `Km_back × Vbus` → 真实电压量纲(V)
- 加死区畸变电压 `v_alpha_dist / v_beta_dist`
- V_DEADTIME = 0.3V（可根据实测标定调整）

### 问题 4：磁链幅值补偿使用前一周期误差 ✅ 已修复

**原问题**: 补偿用旧 Err，滞后一周期。

**修复**: Step 1 先算 ψr，Step 2 算 Err（同周期），Step 5 用同周期 Err 补偿。

### 问题 5：VF→Sensorless 切换后 PLL 增益没恢复 ✅ 已修复

**修复** (`flux.c` → `Observer_Run` Step 8):
- `pll_ramp_active` 标志激活后，每周期向目标增益恢复 2%
- 约 100 周期（5ms）恢复到正常值

### 额外修复：Hybrid Active Flux 观测器 ✅ 2026-07-08

- `OBS_HYBRID_ACTIVE_FLUX`: 按二阶系统 (ζ=1) 带宽设计 PLL
- 反馈补偿 PI (Comp_Kp/Ki)
- 预留 SMO 滑模参数备用

## 1. 先搞清楚"观测器进入"有几种情况

看按键逻辑（`taskManager.c:70`），模式循环是：

```
STOP(0) → VF(3) → SENSORLESS(2) → PREPOS(4) → STOP(0) → ...
```

所以 **Mode 2（无感 FOC）可以从多个路径进入**，观测器的初始状态完全不同：

| 进入路径 | 观测器之前的状态 | 电机状态 |
|----------|-----------------|---------|
| STOP(0) → SENSORLESS(2) | 全部为 0，从未运行 | 静止 |
| VF(3) → SENSORLESS(2) | 全部为 0，从未运行 | **正在 VF 驱动下旋转** |
| PREPOS(4) → 自动跳 SENSORLESS(2) | 已在后台预热运行 | VF 驱动旋转中 |

**最关键的问题**：前两种路径下，观测器的磁链积分器、PLL 全部是初始值 0，**没有任何预热**，直接就要在上岗工作。

## 2. 🔴 观测器算法本身的问题

### 问题 1：PLL 积分器没有抗饱和（Anti-Windup）

**文件**: `Flux/src/flux.c:102-103`

```c
Foc_observer.PLL_Interg += Foc_observer.PLL_Err * Foc_observer.PLL_ki;
Foc_observer.PLL_Ui = Foc_observer.PLL_Err * Foc_observer.PLL_kp + Foc_observer.PLL_Interg;
```

`PLL_Interg` **没有任何上限限制**。如果 PLL 误差长时间很大（比如磁链还没收敛时），积分项会一直累加到一个极大的值。等到磁链终于收敛了，这个巨大的积分项需要很长时间才能"放掉"，期间 PLL 输出完全错误，角度飞转。

**人话**：就像油门卡住了，踩下去弹不回来。PLL 误差大 → 积分器疯狂累加 → 角度飞了 → 电流失控爆鸣。

**修复**：给 `PLL_Interg` 加限幅，例如限制在 `±(2π × 最大电频率 × Ts)` 范围内，或者直接限幅 PLL_Ui：

```c
float ui_max = 2.0f * PI * 400.0f * Foc_observer.Ctrl_ts;  // 400Hz 电频率对应角速度/周期
Foc_observer.PLL_Interg = Limit_Sat(Foc_observer.PLL_Interg, ui_max, -ui_max);
```

### 问题 2：从零冷启动，观测器需要上百毫秒才能收敛

**场景：VF(3) → SENSORLESS(2)，电机正在旋转**

此时：
- `Flux_in_wb = {0, 0}` → 磁链从零开始积分
- `Foc_observer.Theta = 0` → PLL 角度也从零开始
- 电机可能正在 20Hz（约 46 RPM）下旋转

观测器的磁链是靠 **每个周期积分反电动势** 来建立的：

```
Δψ_per_cycle = (Vαβ - Rs × Iαβ) × Ts
             ≈ (jωe × ψf) × Ts     ← 反电动势部分
             ≈ 2π × f × ψf × Ts
             ≈ 2π × 20 × 0.00752 × 0.00005
             ≈ 4.7 × 10⁻⁵ Wb/周期
```

要积累到参考磁链 0.00752 Wb，大约需要 0.00752 / 4.7e-5 ≈ **160 个周期 = 8ms**。

但这是理想情况。实际上：
- 一开始磁链是 0，反电动势也是 0（电机虽然在转，但观测器"以为"转子没动）
- 磁链要从 0 长到参考值，还需要磁链幅值补偿帮忙
- 补偿增益 15000 时，从 0 收敛到参考值大约需要 **几千到上万个周期（几百毫秒）**

**关键**：在这几百毫秒的收敛期内，PLL 锁在一个错误的磁链向量上，输出的角度完全是错的。FOC 电流环用错误的角度做 Park 变换 → Id/Iq 算错 → 输出错误电压 → 电流暴走。

**人话**：观测器就像刚睡醒的人，需要一段时间才能看清东西。代码没给它这个时间，一睁眼就让开车，不出事才怪。

### 问题 3：低转速时逆变器死区造成的电压误差被忽略

观测器用的是 **软件命令的电压**（`Svpwm_dq.Ualpha/Ubeta`），不是电机端子上的 **真实电压**。

真实电压 = 命令电压 − 死区损失 − 开关管压降 − 直流母线纹波

对于 20kHz PWM、死区约 1μs 的情况：
```
死区电压损失 ≈ (死区时间 / PWM周期) × 母线电压 × sign(电流)
              ≈ (1µs / 50µs) × 24V
              ≈ 0.48V
```

在低频轻载时，命令电压可能只有 0.5V～1V，死区误差占了 **50%～100%**！

观测器拿着错误的电压做积分 → 磁链算错 → 角度算错 → 后续全错。

**这是电压模型观测器的固有缺陷**，在低频时尤其严重。工业上通常需要加死区补偿。

### 问题 4：磁链幅值补偿使用前一周期误差（一周期延迟）

**文件**: `Flux/src/flux.c:72`

```c
// 第 72 行：用的 Err 是上一周期第 91 行算出来的
VoltdPhi[0] += FluxR_in_wb[0] * Foc_observer.Err * Foc_observer.Gain;
...
// 第 91 行：本周期才更新 Err
Foc_observer.Err = Foc_observer.Flux * Foc_observer.Flux - g_fluxfluxR;
```

补偿回路有一周期延迟。当 Gain 较大时，这个延迟可能导致 **补偿环路震荡**——磁链幅值在参考值附近来回摆动，PLL 跟踪的也是一个"忽大忽小"的磁链向量。

**人话**：就像洗澡时调水温，你转水龙头后要等 1 秒水温才变，结果每次调过头，水温一冷一热来回震荡。观测器的磁链长度也在参考值附近来回摆，PLL 跟着晃。

### 问题 5：VF→Sensorless 切换后 PLL 增益没恢复

**文件**: `UserSoftware/src/motorControl.c:47-48`

```c
// VF_to_Sensorless_Sync() 中：
Foc_observer.PLL_kp = 1.5f;    // 初始化时是 20
Foc_observer.PLL_ki = 0.08f;   // 初始化时是 10
```

这个降低只发生在 Mode 4 的自动切换路径。如果是按键手动切 Mode 3→2，不会触发这个函数——但也意味着 **PLL 角度和频率完全没有同步到 VF 当前状态**，直接冷启动（问题 2 的情况）。

如果走 Mode 4 路径，增益被降了 13～125 倍，且 **Mode 2 里没有任何代码恢复增益**，PLL 永远在高延迟状态下运行。

### 建议增加的 Vofa 上报通道（观测器调试专用）

建议把 `HFPeriod_RUN` 改成发送以下数据（或者新增一个调试数据包）：

| 优先级 | 变量 | 含义 | 正常时应该长什么样 |
|--------|------|------|-------------------|
| **必看** | `Foc_observer.PLL_Err` | PLL 角度误差 | 接近 0，无明显震荡 |
| **必看** | `Foc_observer.Err` | 磁链幅值误差 | 接近 0，收敛后 < 1e-6 |
| **必看** | `Foc_observer.speed_hz` | 观测器估算电频率 | 应该接近 VF 设定的频率 |
| **必看** | `FluxR_in_wb[0]` | α 轴转子磁链 | 应该是正弦波 |
| **必看** | `FluxR_in_wb[1]` | β 轴转子磁链 | 应该是余弦波（与 α 差 90°）|
| 建议 | `Foc_observer.PLL_Ui` | PLL 输出角速度 | 应该稳定，对应电频率 |
| 建议 | `Foc_observer.Theta` | PLL 估算电角度(rad) | 应该是锯齿波 0→2π |
| 建议 | `PARK_PCurr.Ds` | d 轴电流 Id | 应该≈0（Id=0 控制）|
| 建议 | `PARK_PCurr.Qs` | q 轴电流 Iq | 对应转矩电流 |
| 建议 | `motor.V_d` | d 轴输出电压 | 数值小 |
| 建议 | `motor.V_q` | q 轴输出电压 | 对应转速 |
| 参考 | `Foc_observer.PLL_Interg` | PLL 积分项 | 稳定，不漂移 |

### 调参时重点关注的关系

```
正常运行时应该满足:
  FluxR_in_wb[0]² + FluxR_in_wb[1]² ≈ MOTOR_FLUX²  (磁链幅值恒定)
  PLL_Err ≈ 0                                        (PLL 锁定了)
  speed_hz ≈ CurrentHz                               (速度一致)
  Id ≈ 0, Iq 稳定                                    (电流受控)
```

**如果上面任何一条不满足，观测器就没正常工作。**

## 5. 🎯 推荐的 Debug 步骤

### 步骤 1：先在 VF 模式下跑稳，用 Vofa 看波形

1. 用按键切到 Mode 3（VF），设置 TargetHz=20Hz
2. 电机转起来后，在 Vofa 上观察三相电流波形是否正弦
3. 如果 VF 都跑不稳（电流波形畸变），先解决硬件/死区/SVPWM 的问题

### 步骤 2：扩大 Vofa 上报，看观测器原始数据

修改 `HFPeriod_RUN`，至少加入：
- `Foc_observer.PLL_Err`
- `Foc_observer.Err`
- `FluxR_in_wb[0]` 和 `FluxR_in_wb[1]`

然后在 **VF 模式运行时同步后台跑一下观测器**（但不切过去），在 Vofa 上看：
- `FluxR_in_wb[0] / FluxR_in_wb[1]` 的 XY 图（李萨如图）是不是一个圆
- `PLL_Err` 是否在震荡
- `Err`（磁链幅值误差）是否在收敛

**这一步可以判断观测器算法本身有没有问题**，和切换逻辑无关。

### 步骤 3：调节观测器参数（在后台模式下调）

在 VF 恒定频率（比如 20Hz）下，后台跑观测器，调节：
1. 先调 **Gain**：从 1500 开始，每次 ×2，看磁链幅值什么时候收敛稳定不出震荡
2. 再调 **PLL_kp/PLL_ki**：从 5/2 开始，看 PLL_Err 能否收敛到接近 0
3. 确认 **MOTOR_RS** 是否正确（用万用表实测）

### 步骤 4：尝试切换

观测器在后台收敛好了（Err ≈ 0, PLL_Err ≈ 0），再触发切换。切换时：
- 观测 **不要降 PLL 增益**（把 VF_to_Sensorless_Sync 里降增益那两行注释掉）
- 或者切过去后 **逐步恢复增益**
- 看切换瞬间电流有没有跳变

### 步骤 5：如果还是不行

很可能需要：
- **加死区补偿**：根据电流极性补偿 PWM 占空比
- **提高切换频率到 15-20Hz**：让反电动势足够大
- **考虑混合观测器**：低频用高频注入，高频用反电动势模型

| 问题 | 严重度 | 一句话 |
|------|--------|--------|
| VF→2 直接切换，观测器冷启动 | 🔴 致命 | 磁链从零开始，几百 ms 内全是错的 |
| PLL 积分器无抗饱和 | 🔴 严重 | 误差大时积分爆炸，角度飞转 |
| 无死区补偿 | 🔴 严重 | 低频时电压误差 50%+ |
| 磁链补偿一周期延迟 | 🟡 中等 | Gain 大时可能震荡 |
| Mode 4 切换后 PLL 增益不恢复 | 🟡 中等 | PLL 反应迟钝 |

**最大的疑点**：如果你是通过按键从 VF(3) 直接切 SENSORLESS(2)，观测器完全冷启动在旋转的电机上，这就是爆鸣最可能的原因。**应该走 Mode 4（PREPOS）让观测器先预热再自动切换。**
