## 🔴 P0 — 逻辑错误，影响运行

### 1. Mode 3 (VF) 中多余调用了 `Angel_Get()`

- **文件**: `motorControl.c:57-59`
- **问题**: VF 是开环控制，自己用 `OpenTheta` 积分做角度。在 VF 模式下调用 `Angel_Get()` 会跑一遍完整的磁链观测器 + PLL，结果却没人用（VF 的 iPark 用的是 `ParkSinCos` 而不是 `Park_SinCos`），白白浪费 CPU 且观测器在 VF 下可能发散。
- **修复**: 删除 Mode 3 中的 `Angel_Get()` 调用。

### 2. LCD 显示的模式标签与实际不对应

- **文件**: `taskManager.c:68-84`
- **问题**: LCD 的 Task_DEBUG 显示的模式名称与 `motorControl.c` 实际模式不一致:

  | motorControl.c | LCD 显示 | 正确标签 |
  |---------------|----------|----------|
  | Mode 1: 有感 FOC | `[READY]` | 应为 `[SENSOR]` |
  | Mode 2: 无感 FOC | `[SENSOR]` | 应为 `[SENSORLESS]` |
  | Mode 3: VF 开环 | `[SENSORLESS]` | 应为 `[VF]` |
  | Mode 4: 预定位 | `[VF]` | 应为 `[PREPOS]` |

- **修复**: 对齐标签，或者统一 motorControl.c 和 LCD 的模式编号。

### 3. `key_bsp.h` 中 key_5 与 key_1 引脚冲突

- **文件**: `Key/key_bsp.h:20-21`
- **问题**:
  ```c
  #define key_1 key_bsp_read_pin(GPIOC, GPIO_PIN_9)   // PC9
  #define key_5 key_bsp_read_pin(GPIOC, GPIO_PIN_9)   // PC9 ← 重复！
  ```
  同时 `key_bsp.c` 的 `key_pins[]` 数组只有 4 个元素但定义了 6 个 key 宏。key_5/key_6 未在 `key_check()` 中使用，建议清理或修正引脚。

## 🟢 P2 — 代码质量

### 8. PI 控制器的浮点相等比较不可靠

- **文件**: `PI_Cale.c:7`
- **问题**: `(pV->Out == pV->v1)` 在浮点数中几乎不可能精确命中。积分分离的判断应该改为检查是否触发了限幅:
  ```c
  pV->ui = (pV->Out >= pV->Umax || pV->Out <= pV->Umin) 
           ? pV->i1                    // 已饱和，停止积分
           : pV->Ki * pV->up + pV->i1; // 未饱和，正常积分
  ```

### 9. 速度计算中的魔数 650

- **文件**: `flux.c:114` — `speed_now = speed_acc * 650.0f;`
- **推导**: 期望 speed_acc 累加 8 次 PLL_Ui 后，speed_hz = 平均 ωe / (2π)
  - 8 × 0.0005s × 650 × 2π = 8 × 0.0005 × 650 × 6.283 ≈ 16.33 ≠ 1
  - 正确系数应为 `1.0f / (8.0f * 0.0005f * 2.0f * PI)` ≈ **39.79**
- **修复**: 用计算值替换 650，或者在注释中说明标定依据。

### 10. `Ctrl_ts = 0.0005f` 可能与实际不一致

- **文件**: `flux.c:21` — 硬编码 0.0005 (对应 2kHz?)
- 如果 PWM 频率是 20kHz，一个 PWM 周期 = 0.00005s。注释写的是"20K"，但值却是 0.0005。需要确认实际控制周期并修正。

### 11. `Foc.h` 注释乱码 + 冗余字段

- **文件**: `Foc_Control/inc/Foc.h`
- 注释显示为上世纪的 GB2312 乱码，可读性为零
- 被注释的冗余成员 (`I_d_fb`, `I_q_fb`, `IQAngle_JZ`) 可清理
- `Control_Mode` 注释写 4 个模式但实际用了 5 个 (0~4)

### 12. `key_driver.c` 多处重复代码

- `key_array_init()` 中 4 个 key 的初始化完全相同，可用 `for` 循环替代 50+ 行重复代码
- `key_check()` 中 4 个 key 逐个调用 `hold_filter`，也是复制粘贴
